## 3 — module_shell.html Current State

**887 lines total.**

### postMessage bridge
`window.addEventListener("message", ...)` at **line 665**. Filters on `e.data.type !== "params"` — only processes that one type; everything else silently dropped.

**⚠ Two debug `console.log` lines are live in production code:**
- Line 653: `console.log("[shell] → getParams attempt", attempts, frame.src)` — fires every retry tick
- Line 666: `console.log("[shell] msg:", e.data?.type, e.origin, e.data)` — fires on every incoming message before the filter

These are development artifacts that should be removed before any final release.

### requestParams retry logic
- `MAX = 12`, `BASE = 300 ms`, `CAP = 3000 ms` — exponential backoff
- **50 ms initial delay** before first attempt (`setTimeout(try_, 50)`)
- **`frame.src` guard** at line 646 — skips if iframe has no src yet
- Cancels on a successful `params` response and on each new `load` event

### Hover bridge
`.shellTrigger`: `position:fixed; right:12px; top:148px; width:116px; height:calc(100vh - 168px); z-index:89; pointer-events:auto`

`.shellControls.shown`: adds `opacity:1; pointer-events:auto; transform:translateX(0)`. Base state is hidden + `pointer-events:none`.

JS (lines 805–829): `mouseenter` on trigger → `showControls()` (adds `.shown`, disables trigger pointer-events to let panel buttons receive clicks). `mouseleave` on trigger or controls → `scheduleHide()` with 200 ms timer. `mouseenter` on controls cancels the timer. No CSS `:hover` fallback remaining.

### Nav pills
**Always visible** — `.navPill` has no `opacity:0` base state and the `navTrigger*` divs have no JS reading them. Pills display statically at `top:65px`, `left/right:18px`. Either intentional always-visible chrome, or an unfinished hover-reveal.

The **utility bar** still uses pure CSS `:hover` (not migrated to JS like the shell controls).

Lives as the last child of `aside.shellControls`, correctly filling remaining sidebar height. ## State of the Repo

**Moderate shape — core pipeline is solid, edges are drifting.**

The manifest, shell, and 55-module pipeline are coherent and git is clean. The postMessage bridge pattern is well-designed and correctly implemented in the three wired modules, and the shell's hover/retry architecture is production-quality.

Drift sits in five areas:

1. **52 of 55 modules are unbridge** — the param panel only works for 3 modules; this is the main outstanding milestone.
2. **Three orphaned variant folders** (`brutalist_massing_folded/mat/megaform`) sit at repo root looking like active modules.
3. **Two debug `console.log` calls** remain live in `module_shell.html` (lines 653, 666).
4. **`triangulated_signal_mesh` uses `!important`** in its native UI suppression, violating the migration checklist's own rule.
5. **Housekeeping debt:** accidental absolute-path clone in `_legacy_planning_docs/`, 2.7 MB zip alongside its unzipped copy, large uncompressed thumbnails tracked in git, `.gitignore` with a dead rule and a duplicate entry, stray `make_contact_sheets.py` at root.

None of this blocks work. A cleanup pass before the bridge rollout to the remaining 52 modules would reduce noise and make the migration more tractable.
