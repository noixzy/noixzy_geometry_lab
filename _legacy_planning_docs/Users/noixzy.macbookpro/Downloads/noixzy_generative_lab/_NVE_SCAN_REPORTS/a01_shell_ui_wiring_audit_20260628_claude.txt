SECTION 8 — GENERATED-MODULE CONVERSION RISKS
================================================================================

RISK 1 — SEED vs RANDOMIZE CONFLATION (MEDIUM)
  Generated modules expose "randomize" but not a separate seed-only action.
  Shell "seed" (new seed, same form) and "rnd" (randomize all) both call
  "randomize" → same outcome. No way to advance seed without changing form.
  This breaks the shell contract where seed and rndAll are distinct.
  Recommended fix: generated modules should expose a getSeed/setSeed API or
  a "newSeed" button that only re-seeds without full randomization.

RISK 2 — NO COPY/PASTE STATE IN GENERATED MODULES (LOW)
  Generated modules correctly report copy/paste as unsupported.
  When these modules mature, they will need a state serialization format.

RISK 3 — SHARED RUNTIME COUPLING (MEDIUM)
  noixzy_triangulated_signal_mesh delegates to noixzy_batch_15_generated_runtime.js

  POTENTIAL BROKEN PATH. Verify this file exists in the triangulated_signal_mesh

SECTION 9 — PROCESSOR LOAD RISKS
================================================================================

RISK 1 — NO PAUSE-ON-BLUR IN MODULES (MEDIUM)
  edge_extrude runs p5.js at 60fps.
  sdf_raymarch runs GLSL ray-marching — heavier per-frame cost.
  Neither module appears to throttle when the iframe loses focus or when
  the user is inactive. Only one module is active at a time (single iframe),
  so concurrent load is not an issue. But the active module runs at full GPU
  load continuously until manually paused.
  Recommendation for generated modules: add document.addEventListener("visibilitychange")
  to auto-pause when hidden.

RISK 2 — SDF RAYMARCH AO + 90 MARCH STEPS (LOW-MEDIUM)
  The GLSL fragment shader runs 90 march steps + 5-step AO per fragment at
  full resolution. At 1440p or 2560p this can be GPU-intensive.
  No resolution limiting is visible. Low risk on modern hardware.
  Flag for future: add a resolution/quality toggle.

RISK 3 — IFRAME COLD LOAD ON MODULE SWITCH (LOW)
  Switching modules resets iframe.src. Each module reloads its scripts
  (p5.js CDN, etc.) on every switch. No caching concern but latency is
  noticeable on slow connections. Not a processor issue, a UX issue.
