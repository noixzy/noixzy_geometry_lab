## 2. Recommended Controls for Generated Modules (via build_lab.js)

Generator already injects via SHARED: `zoom`, `rot`, `mirror`, `speed`, `drift`, `metallic`, `rough`, `sheen`, `alpha`, `contrast`, `vig`, `grain`, `glow`.

**Missing — add these:**

| Param | Key | Range | Default | Group | Notes |
|---|---|---|---|---|---|
| center X | `cx` | -1 → 1 | 0 | `frame` | horizontal pan of the render viewport |
| center Y | `cy` | -1 → 1 | 0 | `frame` | vertical pan (positive = up) |

`zoom` + `cx` + `cy` gives full 2D framing. `rot` already exists. That's the complete set — no FOV needed for 2D canvas.

**Implementation in build_lab.js:**  
The frame transform in `draw()` already handles zoom and rot. Add pan by changing:

```js
translate(W/2, H/2)
```
to:
```js
translate(W/2 + P.cx * W * 0.45, H/2 - P.cy * H * 0.45)
```

(Note: invert cy so positive = up, consistent with standard coordinate convention.)

## 3. Recommended Controls for SDF/Raymarch Modules

All five SDF modules share the orbit camera pattern:

```glsl
vec3 ro = vec3(sin(a) * RADIUS, ELEVATION, cos(a) * RADIUS);
```

**Controls to expose per module:**

| Param | Uniform | Range | Default | What it does |
|---|---|---|---|---|
| distance | `u_dist` | 0 → 1 (maps to R_min–R_max) | 0.5 | orbit radius — dolly in/out |
| elevation | `u_elev` | 0 → 1 (maps to 0°–~80°) | 0.4–0.5 | vertical camera angle |

`spin` is already a slider on all five. Together: orbit angle (auto via spin) + orbit radius (distance) + vertical angle (elevation) = complete orbit camera with no added complexity.

**Defer for later:**
- FOV — requires shader recompile path per module, composition varies
- Target pan — all five look at `vec3(0)` or a computed centroid; user-moveable target risks flying off into black space

## 4. Safest First Implementation Pass

### Phase 1 — Generated modules: add cx / cy (one file: build_lab.js)

1. Add to SHARED params array:
   ```js
   {k:"cx", label:"center x", g:"frame", min:-1, max:1, step:0.01, v:0},
   {k:"cy", label:"center y", g:"frame", min:-1, max:1, step:0.01, v:0},
   ```
2. Apply in `draw()`:
   ```js
   translate(W/2 + P.cx*W*0.45, H/2 - P.cy*H*0.45)
   ```
3. Run `node build_lab.js` — all 9 modules regenerate
4. Verify in browser: flow_field and truchet first (simplest), then reaction_diffusion (has a simulation buffer — confirm it's unaffected)
5. Mirror to `~/noixzy_generative_lab/`

This is a 3-line change. Risk is very low — cx/cy default to 0, all existing behavior is unchanged.

---

### Phase 2 — SDF modules: add distance + elevation (5 files, one at a time)

**Order:** gyroid → displacement → mandelbulb → fold → sdf_raymarch  
(leave sdf_raymarch last — it has the most complex JS/preset system)

For each module:
1. Add `u_dist` and `u_elev` uniforms to the fragment shader
2. Replace hardcoded `RADIUS` and `EL` constants:
   ```glsl
   float R  = mix(R_min, R_max, u_dist);
   float el = mix(0.2, 1.1, u_elev);
   ```
3. Add sliders to the panel HTML
4. Add `setUniform` calls in the JS `draw()` function
5. Verify in browser before moving to next module

## 5. Manual-Only Fallback Plan

If no agent is available:

**Safe to do manually:**
- cx/cy in build_lab.js (Phase 1 above) — small, localized, defaults to 0 so zero regression risk
- distance + elevation on ONE SDF module (gyroid recommended) — two shader lines + two HTML rows + two JS lines, all in one file

**Wait for an agent:**
- All 5 SDF modules in one pass — too much copy-paste error surface across files
- sdf_raymarch camera changes — the JS is long and the lock/preset system is complex; easy to break preset serialization
- Any refactor that touches the SHARED panel HTML template string in build_lab.js — one unclosed tag breaks all 9 modules at once

**Rule:** Do one module, verify it works in browser (open → drag slider → see geometry change → save PNG), then proceed to the next. Never edit two SDF modules in the same session without a browser check between them.

## 7. Risks

| Risk | Severity | Mitigation |
|---|---|---|
| cx/cy applied before or after zoom/rot — wrong transform order | Medium | In draw(): `translate(pan) → scale(zoom) → rotate(rot)`. Pan must come before scale or it becomes zoom-relative. |
| SDF orbit radius too small → camera inside geometry → all black | Low | Clamp minimum: map range should never allow R < 1.5× the object's bounding radius |
| SDF elevation near 90° → up vector collapse → `cross(vec3(0,1,0), fw)` = zero → NaN | Medium | Clamp elevation max to 0.9 (≈80°), or use `vec3(0.001, 1.0, 0.0)` as the up vector |
| build_lab.js SHARED syntax error → all 9 modules break at once | Medium | Run `node build_lab.js` immediately after any edit; fix before browser testing |
| sdf_raymarch preset format breaks with new params | Low-Medium | New params with defaults won't break old presets unless the deserializer is strict — verify after adding |
