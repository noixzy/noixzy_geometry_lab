# noixzy generative lab — handoff

_Last updated: 2026-06-23_

A small lab of self-contained algorithmic-art modules for noixzy production (Blender scenes,
music visuals, stills, textures, references). Each module is one HTML file that opens by
double-click (p5.js / WebGL from CDN — needs internet for the CDN).

## Controls (every module)

Grouped, collapsible panel, pinned top-right, translucent over a full-bleed canvas. Press **`h`** to hide it.

- **system** — the module's own 4 controls + a 10-palette selector
- **material** — metallic, roughness, sheen, opacity
- **depth** — **extrude** (2.5D offset depth), **displace** (noise/wave warp)
- **frame** — zoom, rotate, symmetry (2- & 4-fold)
- **look** — contrast, vignette, grain, glow
- **motion** — speed, drift (for loops)

Plus: seed + **new seed**, **★ pin** (save a state), **reset**, **pause**, **save png**,
**export ★** (copy pinned favorites as JSON). Favorites persist in localStorage.

> Note: `grid_extrude` calls its extrude group height/variation/light (real per-cell extrude);
> `sdf_raymarch` has its own 7-slider panel (extrude = Y-stretch, displace = ripple).

## How to work on it

- **Change an engine module or a shared layer:** edit `build_lab.js`, then `node build_lab.js`
  (regenerates all 9). Never hand-edit the generated piece HTML — it gets overwritten.
- **Change a flagship:** edit `grid_extrude/` or `sdf_raymarch/` HTML directly.
- **Refresh thumbnails:** `node contact_sheet.js`.
- **After any change:** copy the touched files into `~/Downloads/noixzy_generative_lab/`.

## Lab → Blender pipeline (proven)

- **reaction_diffusion / cellular_erosion** → grayscale → **displacement map** on a subdivided plane.
  Generator: `cellular_erosion/make_displacement_map.js` (`node … <seed> <size>`).
- **voronoi** → **scatter field** via Geometry Nodes (density from the cell map).
- **sdf** forms → **metaballs** in Blender (smooth-union = the same smin op).
- Renders go to the SSD: `/Volumes/noixzy T5 EVO SSD/ASSET LIBRARY/______newDemo Renders` (1080p).

## Known caveats / decisions

- Engine modules are **static** (pure seed+params); motion comes from the shared transform
  animating a baked buffer (cheap, 60fps) — not per-frame re-simulation.
- Post-processing is **GPU-composited** (canvas filters + overlays), not a CPU pixel loop.
- **extrude** on full-coverage field modules (voronoi, reaction_diffusion) is subtle — the
  offset-stamp needs negative space to read as depth. True volumetric extrude would be a
  per-module geometry rebuild (not done yet).
- UI binds in `setup()`, never `DOMContentLoaded` (that silently killed controls once).
- Thumbnails must NOT use `loading="lazy"` (breaks in backgrounded tabs).
