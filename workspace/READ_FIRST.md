## What this is

A lab of **self-contained algorithmic-art modules** (p5.js, single HTML files that open by
double-click) feeding noixzy production — Blender scenes, music visuals, stills, textures. 11
modules today, an expansion roadmap, and a unified-shell UI in progress.

## 🔴 Non-negotiables (breaking these breaks the lab)

1. **Edit the generator, not the output.** The 9 "engine" modules are GENERATED by `build_lab.js`.
   Change `build_lab.js`, then run `node build_lab.js`. Never hand-edit generated module HTML — it
   gets overwritten. `grid_extrude` and `sdf_raymarch` are hand-authored — edit those directly.
2. **Self-contained always.** CDN `<script>` deps only; modules open by double-click. No npm, no
   bundler, no build step beyond `node build_lab.js`. No new dependencies.
3. **UI binds in `setup()`, never `DOMContentLoaded`** (that silently kills controls).
4. **Shared features go in the engine first**, then replicate into the two flagships.
5. **Verify in a real browser before declaring done** — `python3 -m http.server`, load, reload,
   confirm it renders AND controls respond AND console is clean. A clean parse is not proof.
6. **Mirror every changed file** into `~/Downloads/noixzy_generative_lab/` (same relative paths).
   Refresh thumbnails if visuals changed: `node contact_sheet.js`.
7. **Work additively** — never regress existing params, pins, reset, palettes, save png, or the
   full-bleed translucent UI.

## 🔴 Working conventions (noixzy house rules)

- noixzy is **always lowercase**. Art-first production language — never "tech guy in a cinematic
  room." Restrained, moody.
- Author credit in any script/file header: **"Chris Tucker" only.**
- Deliver scripts as downloadable **`.zip`** with an explicit download step + one chained Terminal
  command that unzips from `~/Downloads` and runs. Don't expect pasted-in script text.
- Before any command, label the **exact app/location** (Terminal, VS Code, Blender, Finder, Browser).
- One Terminal command at a time for file ops; exact filenames; verify with a follow-up; no `cp`
  substituted for `mv`.
- No README files unless necessary. Mark actionable lines with 🔴.
