Do these **in order**, top to bottom. Each task is independent — if you run out of time/credits,
stop after whatever task is fully finished and verified. Don't start a task you can't finish.

## Read first
- `workspace/HANDOFF.md` — full context (architecture, modules, how to run).
- The 9 "engine" modules are GENERATED by `build_lab.js`. **Edit the generator, not the
  generated HTML.** After any engine change: `node build_lab.js`. `grid_extrude` and

## Rules of engagement (apply to every task)
- Keep modules **self-contained** (only CDN `<script>` deps; they open by double-click). No
  build step beyond `node build_lab.js`. No npm install / no bundler.
- Shared features go in the **engine** (so all 9 inherit), then replicate into `grid_extrude`
  and `sdf_raymarch`.
- UI binds inside `setup()`, never `DOMContentLoaded` (that silently breaks controls).
- **Verify in a real browser** before declaring a task done: start a static server
  (`python3 -m http.server`), load the module, reload, confirm it renders AND controls
  respond AND no console errors. A clean parse is not proof.
- After each finished task, copy the changed files into `~/Downloads/noixzy_generative_lab/`
  (mirror of the same relative paths). Commit if the repo is git-tracked.
- Work additively — don't regress existing params (material/depth/frame/look/motion, pins,
  reset, palettes, full-bleed translucent UI, save png).

## TASK 2 — PBR / map export pack
**Goal:** from the field pieces, export a Blender-ready map set, not just a flat PNG.
**Approach:**
- Add an export button "maps" (next to `save png`) in the engine.
- On click, build from the piece's `sampleField` height grid (reuse Task 1):
  - **height.png** — grayscale height (0..1 → 0..255).
  - **normal.png** — tangent-space normal from the height via Sobel (R=dx, G=dy, B=up),
    encoded 0..255.
  - **albedo.png** — the current color render (existing canvas).
  - (optional) **ao.png** — cheap ambient occlusion from local height min.
- Filenames: `noixzy_<piece>_<seed>_height.png` etc. Use canvas `toBlob`/download.
- For pieces without `sampleField` (truchet/l_system/recursive_grid), derive height from the
  rendered luminance instead, so the button works everywhere.
**Acceptance:** clicking "maps" downloads a coherent height+normal+albedo set; normal map
loads correctly as a Blender normal (looks blue-ish, perturbs lighting the right way).

## TASK 3 — Loop / video export
**Goal:** capture one motion cycle to a video file, dependency-free.
**Approach:**
- Use `canvas.captureStream(60)` + `MediaRecorder` (built-in, no library) → WebM.
- Add a "rec" button: starts recording, runs for one loop period, stops, downloads
  `noixzy_<piece>_<seed>.webm`. Loop period: for `sdf_raymarch` = `2π / spin`; for engine
  pieces with `speed>0` use the auto-rotate period; otherwise record a fixed ~6s.
- Don't block the UI; show "rec…" state on the button, restore when done.
**Acceptance:** button produces a playable WebM of a smooth loop. (MP4 is out of scope —
note in a comment that ffmpeg can convert if needed.)
